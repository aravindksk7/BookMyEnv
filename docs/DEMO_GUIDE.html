‚ö†Ô∏è 5. Conflict Detection & Resolution

What is a Conflict?
Conflicts occur when multiple bookings request the same resource during overlapping time periods. The system automatically detects these situations.

Conflict Status Types

Status Meaning
None No conflicts detected for this booking
PotentialConflict Overlapping booking detected, needs review
ConflictConfirmed Conflict verified and pending resolution
Resolved Conflict has been addressed

Resolution Methods

‚úÖ Accept Overlap

Both bookings share the resource with coordination between teams

üîÑ Mark as Resolved

Manually mark as resolved without system changes

‚ùå Remove Resource

Remove the conflicting resource from this booking

‚è∞ Adjust Timing

Change booking start/end times to avoid overlap

üö´ Reject Booking

Cancel the booking due to unresolvable conflict

Resolving a Conflict

  1. Go to Bookings page and click the Conflicts tab
  2. Click the ‚öñÔ∏è Resolve button on the conflicting booking
  3. Review the conflict details and overlapping bookings
  4. Select a resolution method from the options
  5. Add resolution notes for documentation
  6. Click Apply Resolution
